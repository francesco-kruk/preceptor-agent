---
description: Schema and style guide for the learner's WEAKNESSES.md file used by the Preceptor agent
---

`learning/WEAKNESSES.md` is a living document that reflects the learner's **current** gaps — not a permanent archive. It must always conform to this structure.

**Adding entries**: When a new struggle or misconception is identified, append it to the appropriate section.

**Removing entries**: When a learner demonstrably resolves a misconception or overcomes a struggle (i.e. they prove understanding through a correct quiz answer or accurate explanation after a correction loop), **remove that entry** from the file. Do not keep entries that no longer hold true — the file should only contain weaknesses the learner currently has.

```markdown
# Weaknesses

## Struggles
<!-- Topics or skills where the learner has shown repeated difficulty. One bullet per entry. -->
- <topic>: <brief description of the difficulty observed>

## Misconceptions
<!-- Specific wrong beliefs revealed by quiz answers or explanations. One bullet per entry. -->
- <topic>: <exact wrong belief held> (e.g. "Believed that `let` is function-scoped like `var`")
```
